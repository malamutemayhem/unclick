# Web app builder

Use this to turn a short brief into a working, good-looking web app or artifact.

## Run

1. Pin the scope to one screen or flow that proves the idea; cut the rest.
2. Lay out structure and states before styling: data, empty, loading, error, success.
3. Use the project's component library and tokens so it matches the rest of the app.
4. Wire real interactions; do not ship dead buttons or placeholder handlers.
5. Make it responsive and keyboard-usable from the first pass.

## Rule

Working and simple beats broad and broken. Ship one solid flow, then extend.

## Output

Return the files created, how to run it, and what is stubbed versus real.
