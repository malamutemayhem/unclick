It sees shape, not data

By Sam Okafor, 12 June 2026

Most tools that read the web try to extract data. They look for a price, a title, a date, and they scrape it.

That approach is brittle. Change a class name and the scraper breaks. Add a wrapper and the selector misses.

We took a different path. Instead of extracting data, we recognise components. A grid of cards is a grid of cards whether it sells shoes or lists news.

Once you see the component, both outputs fall out of the same observation: a calm human view and an agent readable shape.

The same observation, pooled across users, accretes into a shared index of site shapes. That index is the real product.
